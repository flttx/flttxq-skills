# React 审查清单

适用于 React、Next.js、Vite React 等目录。

## 重点检查

- 组件状态是否重复来源，是否出现派生状态失真。
- `useEffect` 依赖是否正确，是否引入陈旧闭包。
- 事件处理、异步请求、路由切换后是否存在竞态或卸载后更新。
- Context、全局状态、缓存层是否引入额外耦合。
- 列表、重渲染、昂贵计算是否存在明显性能问题。
- SSR/CSR 边界、浏览器 API 使用时机是否正确。

## 常见误区

- 通过 `useEffect` 补业务流程，导致逻辑分散和时序不稳定。
- 用局部 state 镜像 props 或 store 数据。
- 为了“优化”过度引入 memo，反而增加复杂度。
