# 平台适配

同一套审查逻辑，在不同终端上主要调整的是“表达方式”，不是判断标准。

## Codex

- 优先输出发现的问题，按严重级别排序。
- 每条问题尽量短、准、可执行。
- 总结放后面，避免把概览写得比问题更长。

## Claude Code

- 除问题外，更明确地写出假设、未验证范围和证据不足点。
- 当问题存在争议时，补一段 reasoning summary，但不要过长。

## Antigravity

- 先给结论摘要，再列关键问题。
- 更强调结果压缩和管理视角，不要展开过多实现细节。

## 通用适配原则

- 不要因为平台不同就改变严重级别标准。
- 不要输出不带证据的猜测。
- 不要把纯风格问题放到高优先级。
- 没有问题时明确写“未发现问题”，并说明验证缺口。
