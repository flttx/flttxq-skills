# 通用审查基线

这份清单适用于所有项目，进入 code review 时默认先读。

## 1. 行为与正确性

- 变更是否改变了原有调用时机、状态机分支或接口语义。
- 是否存在空值、未初始化值、边界值、默认值错误。
- 是否存在并发、竞态、重复提交、旧响应覆盖新状态。
- 是否引入了只在特定环境、平台、时序下触发的问题。

## 2. 异常与恢复

- 错误是否被吞掉，导致失败不可见。
- 恢复路径是否存在，是否能重试、回退或降级。
- 关键异常分支是否有足够日志支撑排查。

## 3. 安全与隐私

- 是否泄露密钥、令牌、用户隐私数据。
- 是否缺少输入校验、路径校验、权限边界检查。
- 是否存在危险默认值、调试开关误入生产。

## 4. 可维护性

- 是否让职责边界变得更混乱。
- 是否新增了重复逻辑或隐式耦合。
- 是否把复杂逻辑继续堆到巨石文件中。
- 命名、模块划分、注释是否能支撑后续维护。

## 5. 性能与资源

- 是否引入不必要的重复请求、重复渲染、重复计算。
- 是否造成缓存失效、内存泄露、监听未释放、资源未关闭。
- 大对象、列表、图片、文件流是否有明显开销风险。

## 6. 验证完整性

- 改动类型是否匹配了最小必要的 lint、test、build、手工验证。
- 交付说明是否解释了未验证部分与剩余风险。

## 输出要求

每个问题尽量包含：

- 位置
- 触发条件
- 实际影响
- 结论理由
- 最小修复方向或验证方式
