# 项目规范扫描

这份文档用于把“仓库自己的规则”纳入 review，而不是只套通用模板。

## 优先读取的文件

- `AGENTS.md`
- `CLAUDE.md`
- `README.md`
- `docs/**`
- `.cursor/rules/**`
- 与测试、构建、发布、架构、状态机、导航、权限、主题相关的专项文档

## 扫描顺序

1. 先找与当前改动目录同层或父层的规范。
2. 再找仓库根目录规范。
3. 最后找 `docs/**` 中与当前模块强相关的专项文档。

## 需要提炼出的信息

- 文件大小门禁、拆分规则
- 必跑验证命令
- 状态机不变量
- 请求竞态或缓存一致性要求
- 错误日志与诊断上报要求
- 主题、国际化、权限、启动链路等特定约束

## 处理冲突的方法

- 更靠近改动目录的规则优先。
- 更保守的规则优先。
- 如果仍冲突，明确写出冲突点，不要自行忽略。

## 评审时要避免

- 只看框架最佳实践，不看项目已有约束。
- 用通用建议覆盖仓库特定设计。
- 在不了解项目不变量的情况下建议大改结构。
