# 腾讯应用宝审查清单（预审版）

> 本清单按腾讯开放平台“审核规范 + 上架流程 + 隐私专项”组织。  
> 优先规则：`移动应用审核规范(4007776097)`、`隐私政策提交内容及审核规范(875339652)`、`申请上架(4008518303)`。  
> 规范来源总表见 `references/tencent-myapp-source-map.md`。

## 1. 版本可测性与上架流程

对应官方条款：

1. `4008518303` 申请上架
2. `4009007496` 更新版本
3. `4008788830` 审核撤回
4. `4007776097` 移动应用审核规范

检查要求：

1. 安装、启动、主流程、更新路径可正常走通，无高频崩溃或 ANR。
2. 审核账号、验证码、白名单、区域限制等信息在审核说明中完整可复现。
3. 新版本提审前核对“包体能力变化 vs 上架资料”一致性，避免旧资料失配。

## 2. 元数据、内容与展示一致性

对应官方条款：

1. `4007776097` 移动应用审核规范
2. `4007776049` 新广告法审核规范
3. `4007776056` 年龄分级审核规范

检查要求：

1. 应用名称、图标、截图、简介与真实功能一致，不堆砌热词或误导描述。
2. 广告文案、推荐语、运营弹窗不触发夸大宣传和误导点击。
3. 年龄分级与应用内容一致，未成年人保护相关提示不缺失。

## 3. 隐私、权限与账号注销闭环

对应官方条款：

1. `875339652` 隐私政策提交内容及审核规范
2. `4007776457` 隐私政策
3. `4007776059` APP隐私政策规范及隐私检测报告查看入口
4. `4007776069`~`4007776077` 强制/频繁/过度索权与关联启动场景
5. `889011983` 用户隐私权限审核规范
6. `4010287290` 腾讯开放平台账号注销协议

检查要求：

1. 首次进入应用需展示隐私政策弹窗，且用户可明确“同意/拒绝”。
2. 应用内需提供用户协议与隐私政策常驻入口，查询路径不超过 4 步。
3. 权限申请与业务功能逐项对应，拒绝后有降级路径，不得强迫授权。
4. 提供账号注销或数据删除路径，并与隐私政策承诺一致。

## 4. 资质材料与备案一致性

对应官方条款：

1. `885855051` 应用资质材料规范
2. `4008550787` APP备案政策及接入指引
3. `4008450824` ICP备案政策及接入指引
4. `4008450265` 移动应用ICP备案截图上传
5. `4008450352` ICP备案常见问题

检查要求：

1. 包名、应用名、开发者主体、资质材料信息一致且在有效期内。
2. 备案信息与应用类型匹配，截图、编号、主体名称可交叉验证。
3. 新增支付/内容/行业服务能力时，补齐对应专项资质后再提审。

## 5. 违规、处罚与申诉风险

对应官方条款：

1. `4007776131` 移动应用违规说明
2. `4009014011` 违规说明
3. `945512116` 腾讯开放平台应用运营管理办法
4. `4007776130` 应用侵权（投诉/反投诉体系）

检查要求：

1. 排查诱导下载、诱导分享、误导付费、违规收集信息等高危行为。
2. 对“历史处罚项”做回归检查，确认版本已闭环整改后再提审。
3. 若存在侵权风险（名称、图标、内容、素材），先完成授权/替换。

## 高频 P0 拒审信号（优先排查）

1. 首开无隐私弹窗，或弹窗无“拒绝”选项。
2. 隐私政策/用户协议无常驻入口，或入口过深不可达。
3. 权限用途描述与实际调用不一致，存在过度索权。
4. 资质备案缺失或与当前版本能力不匹配。

## 输出建议

每个问题项建议包含：

1. `疑问描述`（附 `directoryId`）
2. `可能触发的拒审原因`
3. `证据`（源码/配置/构建包/提审截图）
4. `建议动作`（含修复顺序）
5. `优先级`（P0/P1/P2/INFO）
