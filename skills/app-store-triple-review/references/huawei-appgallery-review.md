# 华为应用市场审查清单（预审版）

> 本清单按华为官方“审核政策 + 发布流程”组织。  
> 优先规则：`应用审核指南(50104)`、`应用资质审核要求(80301)`、`发布HarmonyOS应用`链路。  
> 规范来源总表见 `references/huawei-appgallery-source-map.md`。

## 1. 基础可用性与审核可复现

对应官方条款：

1. `50104` 应用审核指南（应用功能/稳定性主线）
2. `agc-help-release-app-reviewinfo-0000002320780365` 填写审核信息与联系方式
3. `agc-help-release-app-submit-0000002286180890` 提交审核

检查要求：

1. 安装、启动、登录、核心闭环可稳定复现，不依赖不可访问环境。
2. 审核说明提供可用测试账号/验证码/白名单信息，审核路径可一步步复现。
3. 断网、权限拒绝、登录失效等异常路径可回退，不出现卡死或崩溃主路径。

## 2. 元数据与素材一致性

对应官方条款：

1. `50104-01` 应用信息
2. `agc-help-app-material-requirement-0000001146534651` 应用素材规范
3. `agc-help-app-visual-asset-spec-0000002277607976` 素材规范

检查要求：

1. 应用名称、简介、图标、截图、视频与真实功能一致，不夸大、不蹭热点。
2. 图标/截图清晰、无误导角标或侵权元素，素材尺寸与格式符合平台要求。
3. 上架信息、包体信息、应用内展示信息一致（名称/版本/主功能）。

## 3. 隐私、权限与数据处理

对应官方条款：

1. `50104-07` 用户隐私
2. `privacy-label` AppGallery隐私标签介绍
3. `agc-help-release-app-privacy-state-0000002278878296` 配置隐私声明
4. `agc-help-release-app-privacy-desc-0000002313477969` 配置隐私说明
5. `agc-help-privacy-policy-app-0000002282162168` 配置隐私政策（HarmonyOS应用）

检查要求：

1. 应用内可访问隐私政策，且与实际 SDK/权限采集行为一致。
2. 敏感权限按最小化原则申请；拒绝授权后提供可理解的降级路径。
3. 隐私标签、隐私声明、应用实际行为三者一致，避免“声明有/代码无”或“代码有/声明无”。
4. 账号体系应具备注销与数据处理说明，入口可达、流程可执行。

## 4. 资质、备案与分类合规

对应官方条款：

1. `50104-10` 应用资质
2. `80301` 应用资质审核要求
3. `50111` 应用资质FAQ
4. `50130` APP核准（APP备案）指引
5. `agc-help-release-app-record-0000002319594705` 配置核准（备案）信息

检查要求：

1. 行业资质、软著、备案等与应用能力匹配，提交材料真实且在有效期内。
2. 高监管类别（金融/医疗/新闻/地图等）补齐对应专项资质。
3. 应用分类、标签与实际内容一致，不规避分类或错填备案信息。

## 5. 发布流程与审核状态联动

对应官方条款：

1. `agc-help-release-app-guide-0000002287176372` 发布指引
2. `agc-help-release-app-submit-0000002286180890` 提交审核
3. `agc-help-release-push-0000002312626049` 催促/加急/撤销审核
4. `agc-help-maintain-remove-0000002274058145` 下架应用/元服务
5. `agc-help-maintain-delete-0000002271413701` 删除应用/元服务

检查要求：

1. 提审前完成“包体、隐私、资质、审核信息”四类必填项一致性检查。
2. 更新版本时复核新增权限/SDK/付费路径，防止历史材料失效。
3. 审核驳回后按驳回点补证据，不直接重复提交同一问题包体。

## 6. 内容安全、侵权与处罚风险

对应官方条款：

1. `50104` 应用审核指南（内容、安全、广告、付费、知识产权）
2. `50120` 应用侵权投诉处理指引
3. `50125` 应用年龄分级标准
4. `news` 监管政策与审核策略动态

检查要求：

1. 内容不触发违法违规、低俗、侵权、误导未成年人等高风险项。
2. 广告、付费、会员权益透明可解释，不误导点击或误导收费。
3. 年龄分级、内容标签、应用实际内容保持一致。

## 高频 P0 拒审信号（优先排查）

1. 隐私声明/隐私标签与实际采集行为不一致。
2. 资质或备案缺失，或提交材料与应用能力不匹配。
3. 素材和描述明显误导，或存在侵权风险。
4. 审核账号不可用，审核路径无法复现核心功能。

## 输出建议

每项结论建议包含：

1. `检查点`（附官方条款 ID）
2. `结论`：PASS / PASS_WITH_QUESTIONS / BLOCKED
3. `风险级别`：P0 / P1 / P2 / INFO
4. `证据`：文件路径 / 构建产物 / 操作记录
5. `建议`：可执行修复动作（含优先级与责任模块）
