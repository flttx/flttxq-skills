# 腾讯应用宝审核规范来源矩阵

## 目的

回答“应用宝规范很多，当前到底参考哪几份”的问题，避免把移动应用规则、PC 接入规则、培训文档混为同一优先级。

## 来源优先级（由高到低）

1. `L1 强制审核规则`：移动应用审核/隐私/违规类规范（直接决定是否通过）。
2. `L2 提交流程规则`：申请上架、更新版本、撤回审核等提审流程规范。
3. `L3 时效与政策要求`：备案政策、平台公告、专项治理等时效规则。
4. `L4 辅助说明`：FAQ、培训课程、示例文档（用于解释，不替代 L1/L2）。

## 使用规则

1. 若 L1 与其他层冲突，以 L1 为准。
2. 移动应用与 PC 应用文档冲突时，移动应用路径优先；PC 文档仅作补充说明。
3. 若条款含生效时间（备案/政策专项），报告中必须写明具体日期再出结论。
4. 若结论仅来自 L4 培训/FAQ 文档，标记为 `PASS_WITH_QUESTIONS` 并建议人工复核。

## 当前覆盖边界（需要明确）

1. 游戏类需切换到“移动游戏审核规范”体系，不能直接套用本矩阵。
2. 行业专项（教育、医疗、金融、新闻等）需叠加对应备案和资质规则。
3. SDK 数据采集合规需结合最新隐私检测报告，不仅依赖文本条款。
